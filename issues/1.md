## ⚙️ Add GitHub Actions CI Workflow

Adding a CI pipeline will make the project look more professional and ensure code quality is maintained as contributors submit PRs.

### Tasks
- [ ] Create `.github/workflows/ci.yml`
- [ ] Run `pytest` on every push and pull request
- [ ] Add a Python linting step with `flake8` or `ruff`
- [ ] Add a badge to the README showing CI status

### Expected Outcome
Green CI badge on the README and automated testing on every PR.

**Great for DevOps contributors!** 🚀
